# Quality Attribute Specification

> 본 문서는 `02_requirements.md`에서 도출한 QA-01~QA-09을 품질 속성 명세(Quality Attribute Specification)로 정리한다.
>
> 진행 순서: 요구사항 수집 → 요구사항 도출 → **품질 속성 명세(본 문서)** → Utility Tree 정리 → Architectural Driver 선정

관련 유즈케이스 명세는 [`01_use_case_spec.md`](01_use_case_spec.md) 참조.

## 1. 명세 원칙

| 항목 | 내용 |
|------|------|
| 범위 | Secure Vision AI Platform의 보안, 성능, 확장성, 가용성, 자원 효율, 시험 용이성, 변경 용이성 |
| 입력 | `02_requirements.md`의 QA-01~QA-09 |

| 우선순위 | `03_utility_tree.md`의 리프 시나리오 평가 결과와 동일하게 1~9순위 적용 |
| 표기 | QA 번호는 우선순위 순서와 일치하며, QS 번호는 동일 번호의 품질 시나리오를 뜻함 |

## 3. 상세 명세

### QS-01: 보안 — Host 침해 시 기밀성 (QA-01)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-01, UC-04 |
| 관련 Driver | FR-01, FR-04, CONST-02, CONST-04 |
| 자극원 | Host OS 루트 권한을 탈취한 공격자 |
| 자극 | Host 커널 권한으로 pVM 메모리 읽기 시도 |
| 환경 | Secure Vision AI 파이프라인 정상 동작 중 |
| 대상 | pVM 내 영상 원본, AI 모델 가중치, 추론 중간 데이터 |
| 응답 | Stage-2 격리에 의해 모든 접근이 차단되고, 비정상 접근 시도가 기록된다 |
| 응답 측정치 | 침투 시험 전 케이스에서 격리 메모리 노출 0건 |
| 검증 방법 | `/dev/mem`, KVM 인터페이스 오용, 페이지 테이블 조작 등 공격 벡터를 자동 실행하고 pVM 소유 메모리 노출 여부를 확인 |
| 수용 기준 | 읽힌 바이트 수가 0이어야 하며, 비정상 접근 로그가 남아야 한다 |
| 우선순위 | 1순위 |

### QS-02: 보안 — HW 접근 격리 (QA-02)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-03 |
| 관련 Driver | FR-03, CONST-05 |
| 자극원 | Host 일반 기능과 보안 파이프라인 |
| 자극 | Camera/AI HW 사용 주체 전환 또는 동시 사용 요구 |
| 환경 | SW 중재 기반 HW IP 공유 운용 중 |
| 대상 | HW IP 중재 계층, S2MPU 권한 상태 |
| 응답 | 이전 주체 권한 회수 후 다음 주체 권한을 부여하며, 권한 중첩 구간이 발생하지 않는다 |
| 응답 측정치 | S2MPU 권한 상태 로그에서 권한 중첩 구간 없음 (위반 횟수 = 0) |
| 검증 방법 | 권한 전환 코드 경로에서 회수→부여 순서를 검사하고, 비활성 주체의 HW 레지스터 접근을 주입 |
| 수용 기준 | 권한 중첩 로그가 없어야 하며, 비활성 주체 접근은 S2MPU fault로 차단되어야 한다 |
| 우선순위 | 2순위 |

### QS-03: 성능 — 실시간 처리 (QA-03)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-02, UC-03, UC-04 |
| 관련 Driver | FR-02, FR-03, FR-04, CONST-05 |
| 자극원 | 카메라 센서 |
| 자극 | 1080p 30fps 영상 스트림 지속 유입 |
| 환경 | 격리 파이프라인(Secure Camera→Secure AI) 정상 동작, Host 통상 부하 |
| 대상 | End-to-End 파이프라인(캡처→Camera HW→AI HW 추론→판단) |
| 응답 | Camera/AI HW 가속으로 프레임 드롭 없이 처리 |
| 응답 측정치 | 30fps 유지, 캡처→판단 E2E 지연 100ms 이하, 비격리 구성 대비 처리 성능 저하 10% 이내 |
| 검증 방법 | 동일 HW에서 비격리 baseline과 격리 구성의 fps, E2E 지연, CPU 사용률을 비교 |
| 수용 기준 | 격리 구성의 성능 저하가 baseline 대비 10% 이내여야 한다 |
| 우선순위 | 3순위 |

### QS-04: 확장성 — 신규 Workload 수용 (QA-04)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-05 |
| 관련 Driver | FR-05, CONST-06 |
| 자극원 | 로봇 제조사 / Workload 개발자 |
| 자극 | 신규 보안 Workload 추가 요구 |
| 환경 | Framework가 제품에 배포/운용 중인 상태 |
| 대상 | Framework 본체(Middleware/커널 드라이버) |
| 응답 | 펌웨어 재배포/Framework 수정 없이 Workload 패키징/탑재만으로 수용된다 |
| 응답 측정치 | Framework 코어 수정 0 LoC |
| 검증 방법 | Workload 추가 전후 diff에서 Framework 코어 디렉터리 변경 라인 수를 카운트 |
| 수용 기준 | 코어 변경 라인이 0이어야 하며, 서명 검증된 패키지만 탑재되어야 한다 |
| 우선순위 | 4순위 |

### QS-05: 성능 — 도메인 간 통신 오버헤드 (QA-05)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-04 |
| 관련 Driver | FR-04, QA-01, QA-03 |
| 자극원 | Secure Camera pVM |
| 자극 | 프레임 단위 대용량 영상 데이터를 Secure AI pVM으로 전달 |
| 환경 | 파이프라인 정상 동작 중 |
| 대상 | 도메인 간 보안 채널(공유 메모리/RPC) |
| 응답 | 데이터 노출 없이 전달되며 파이프라인 실시간성이 유지된다 |
| 응답 측정치 | 프레임 데이터 복사 횟수 0회 또는 프레임당 전달 지연 5ms 이하 |
| 검증 방법 | dma-buf 등 버퍼 전달 경로의 복사 여부를 코드로 확인하고 프레임 전달 지연을 계측 |
| 수용 기준 | zero-copy 경로가 유지되거나, 전달 지연이 프레임당 5ms 이하이어야 한다 |
| 우선순위 | 5순위 |

### QS-06: 가용성 — pVM 장애 격리 (QA-06)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-01, UC-02 |
| 관련 Driver | FR-01, FR-02 |
| 자극원 | 오동작하는 보안 Workload |
| 자극 | pVM 비정상 종료(크래시, 무응답) |
| 환경 | 다중 pVM 운용 중인 로봇 통상 동작 상태 |
| 대상 | Host OS, 타 pVM, 로봇 기본 동작 |
| 응답 | 장애가 해당 pVM에 한정되고, Framework가 자원을 안전하게 회수한 뒤 재시작한다 |
| 응답 측정치 | Host/타 pVM 다운타임 0, 장애 pVM 재시작 3초 이내 |
| 검증 방법 | 장애 주입 후 Host/타 pVM 상태와 장애 pVM 재시작 시간을 계측 |
| 수용 기준 | 장애가 다른 도메인으로 전파되지 않고, 서비스 재개가 3초 이내 완료되어야 한다 |
| 우선순위 | 6순위 |

### QS-07: 자원 효율 (QA-07)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-01, UC-02 |
| 관련 Driver | FR-01, FR-02 |
| 자극원 | 시스템 통합자(로봇 제조사) |
| 자극 | 보안 Framework + pVM 2개(Camera, AI) 상시 운용 |
| 환경 | 로봇 제품의 통상 동작 상태 |
| 대상 | SoC 메모리/전력 예산 |
| 응답 | 비격리 구성 대비 추가 자원 소모가 제품 탑재 가능 한도 이내로 유지된다 |
| 응답 측정치 | 추가 메모리 256MB 이하, 전력 증가 5% 이내 |
| 검증 방법 | `/proc/meminfo`, cgroup 집계, PMIC 텔레메트리 또는 전력 계측 장비로 비격리 구성과 비교 |
| 수용 기준 | 메모리와 전력 증가분이 각각 목표 이내여야 한다 |
| 우선순위 | 7순위 |

### QS-08: 시험 용이성 — 격리 보장 검증 (QA-08)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-03, UC-04 |
| 관련 Driver | QA-01, QA-02, QA-05 |
| 자극원 | 품질/검증 조직 |
| 자극 | 주요 격리 보장에 대한 객관적 검증 요구 |
| 환경 | 통합 시험 단계 및 회귀 시험 |
| 대상 | 격리 메커니즘(Stage-2, S2MPU, 보안 채널) |
| 응답 | Host 침해 모사 도구 등 재현 가능한 자동화 시험으로 격리 유지가 검증된다 |
| 응답 측정치 | 주요 격리 요구사항의 자동화 시험 커버 100% |
| 검증 방법 | 공격 벡터 카탈로그와 자동화 테스트 매핑표에서 미커버 항목 수를 카운트하고 CI 반복 실행 여부를 확인 |
| 수용 기준 | 미커버 격리 항목이 0개여야 하며, 회귀 시험이 반복 실행 가능해야 한다 |
| 우선순위 | 8순위 |

### QS-09: 변경 용이성 — Secure OS 교체 (QA-09)

| 요소 | 명세 |
|------|------|
| 관련 UC | UC-06 |
| 관련 Driver | FR-06, CONST-03 |
| 자극원 | 서드파티 Secure OS 벤더 / 로봇 제조사 |
| 자극 | 탑재된 Secure OS를 다른 Secure OS로 교체 |
| 환경 | 설계/통합 단계 또는 제품 유지보수 단계 |

| 검증 방법 | Secure OS 교체 전후 diff에서 Secure OS 패키지 외부 파일 변경 수를 카운트 |
| 수용 기준 | 패키지 외부 변경 파일이 0개여야 하며, 기존 ENC/DEC 기능이 회귀하지 않아야 한다 |
| 우선순위 | 9순위 |

## 4. 품질 속성 간 상충 관계

| 상충 관계 | 관련 QA | 설계 시 확인할 사항 |
|----------|---------|-------------------|
| 보안 vs 성능 | QA-01, QA-02, QA-03, QA-05 | Stage-2/S2MPU/보안 채널 강화가 실시간 경로에 주는 비용을 계측한다. |
| 보안 vs 확장성 | QA-01, QA-02, QA-04 | Workload 동적 탑재가 신뢰 경계와 공격 표면을 넓히지 않도록 서명/권한 정책을 강제한다. |
| 성능 vs 자원 효율 | QA-03, QA-05, QA-07 | 실시간 처리를 위한 버퍼, pVM, HW 예약량이 메모리/전력 예산을 넘지 않는지 확인한다. |
| 변경 용이성 vs 안정성 | QA-06, QA-09 | Secure OS 교체와 pVM 재시작 경로가 기존 TrustZone 기능과 생명주기 관리에 회귀를 만들지 않는지 확인한다. |
