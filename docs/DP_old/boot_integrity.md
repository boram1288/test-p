## 1. 결정 범위와 전제

### 1.1 설계질문 (05 문서 DP-09)

- pVM 이미지와 워크로드 패키지의 서명 검증은 EL2, Host 드라이버, 별도 서비스 pVM, TrustZone TEE 중 누가 수행할 것인가?
- measured boot 체인은 어디서 시작하고 어디서 종료할 것인가?
- EL2 수정 없이(CONST-02) 이미지 진위와 측정값을 신뢰할 수 있게 만드는 방법은 무엇인가?
- 검증 실패 시 pVM 생성, 워크로드 로딩, 채널 수립을 어떻게 차단할 것인가?
- 채널 상대 인증에서 사용할 pVM 아이덴티티에 부팅 측정값을 어떻게 반영할 것인가?

### 1.2 선행 DP 결정이 주는 전제와 역반영 의무

본 DP는 **3건의 역반영 게이트가 수렴하는 지점**이다 — 본 DP의 결정이 곧 다음 게이트들을 닫는다.

- **DP-03 G2 (역반영 대상)**: VMID와 신뢰 아이덴티티의 결합은 본 DP(측정값)(아이덴티티) 완료 후 역반영한다. 그때까지 VMID 단독 인증 금지.
- **채널 상대 인증 설계 (역반영 대상, CRITICAL)**: 측정값↔자격 바인딩의 구체 형태(측정값 포맷·전달 경로·바인딩 시점)는 본 DP가 확정해 역반영한다. 채널 상대 인증 설계 = TEE 발급, measurement-bound 서명, ephemeral 키, private key 미방출.
- **DP-07 G1·G2 (역반영 대상)**: 키 방출 조건 "측정값 대조"의 구체 형태(측정값 포맷·측정 주체·대조 시점)를 본 DP가 확정한다. measured boot 체인 종료 지점(워크로드 코드까지 vs pVM 이미지까지)이 변조 워크로드 차단 범위를 결정한다.
- **DP-02 G5**: measured 권한 바인딩 — manifest 권한과 측정값의 연결은 본 DP의 측정 결과를 입력으로 한다.
- **DP-03**: TEE가 1차 신뢰 앵커, pVM→TEE는 EL2 SMC 트랩·포워딩 경로. 측정값 보관·대조 TA는 G5 예산표 등록 대상.
- **DP-08 (병행)**: 검증 지점은 패키지 형식·설치/로딩/교체 절차(13 문서 공통 설계 요소)와 맞물려 확정한다.
- **CONST-02**: EL2(pKVM) 수정 불가 — pKVM이 **기 제공하는** 초기 측정 기능만 사용 가능하다.

## 2. 후보 구조 (생성: Claude)

### 후보 A: EL2 초기 측정 + pVM 내장 검증 부트로더 체인 (pvmfw/DICE 패턴)

pKVM이 기 제공하는 초기 측정 기능(pVM 최초 메모리 이미지의 해시 측정)으로 **신뢰 부트로더**(프레임워크 제공, 모든 pVM 공통)의 무결성을 EL2가 보증한다. 이 부트로더가 pVM의 첫 코드로 실행되어 다음 단계(P1 베이스/P2 커널 → 워크로드 페이로드)를 서명 검증·측정하고, 단계별 측정값을 누적(DICE 유사 체인)하여 EL2 SMC 경로로 TEE에 보고한다. TEE는 측정값을 보관하고 키 방출(DP-07)·자격 발급(채널 상대 인증) 시 대조한다.

- **체인**: EL2 초기 측정(부트로더) → 부트로더가 게스트 이미지 검증·측정 → 게스트가 워크로드 페이로드 검증·측정(P1은 베이스 런타임이 수행) → TEE 누적 보고. **종료 지점 = 워크로드 코드·자산까지**.
- **장점**: 측정·검증이 pVM 격리 경계 **안**에서 일어나 Host(AT-2)가 개입 불가 — 검증 후 변조(TOCTOU)가 EL2 Stage-2 보호로 차단된다. EL2 수정 불필요(기 제공 측정만 사용). 체인이 워크로드까지 연장되어 DP-07 4.3절 질문("변조 워크로드 차단 범위")에 가장 강하게 답한다.
- **단점**: pKVM의 초기 측정 기 제공 여부·측정 대상 범위를 HV 파트에 확인해야 한다(H 계약 추가 후보). 공통 부트로더가 전 pVM의 TCB가 된다 — 결함 시 전체 영향. 부트로더→TEE 보고 경로의 부트스트랩(자격 없이 측정값 제출)이 DP-06 G3 버퍼 순서와 맞물린다.

### 후보 B: TEE 사전 검증·승인 (이미지 제출형)

pVM 생성 전에 Host(관리 서비스)가 이미지·manifest를 TEE에 제출하고, TEE가 서명 검증·해시 측정 후 승인 기록을 보관한다. 이후 pVM이 부팅하면 TEE는 승인된 측정값 기준으로 키 방출·자격 발급을 판정한다.

- **장점**: 검증 로직이 TEE 한 곳에 집중되어 단순하다. EL2 의존이 전혀 없다.
- **단점**: **TOCTOU 결함** — TEE가 검증한 이미지와 실제 pVM에 로딩된 이미지가 동일함을 보증할 주체가 없다. Host가 검증 후 로딩 전에 이미지를 바꾸면 TEE는 알 수 없다. 실제 로딩 내용의 측정(EL2 또는 pVM 내부)과 결합하지 않으면 AT-2에 무력 — 단독 성립 불가. 대용량 이미지의 TEE 제출은 DP-03 소용량 규칙과도 충돌.

### 후보 C: Host 1차 검증 + EL2 기 제공 측정값 사후 attestation

Host 드라이버가 설치 시 서명을 1차 검증(운영 편의·조기 거부)하고, 실질 보증은 pKVM 기 제공 초기 측정값(pVM 전체 이미지)을 TEE가 사후 대조하는 것으로 한다. 부트로더 체인 없이 초기 이미지 측정 하나로 끝낸다.

- **장점**: 구조가 가장 단순하고 추가 게스트 컴포넌트가 없다. Host 1차 검증은 정상 운영에서 불량 패키지를 조기에 걸러낸다.
- **단점**: 측정 종료 지점이 **초기 이미지까지** — 부팅 후 로딩되는 워크로드 페이로드(2단계 startup의 모델 자산, P1 페이로드)는 측정 범위 밖이다. DP-07의 "변조 워크로드 차단"과 DP-02 G5의 measured 권한 바인딩이 페이로드 수준에서 성립하지 않는다. Host 1차 검증은 AT-2에서 무력(보조일 뿐).

### 공통 설계 요소

- **검증 실패 시 차단(fail-closed starvation)**: EL2 수정 불가로 pVM 부팅 자체의 차단은 보장하지 못할 수 있다 — 대신 측정값 대조 실패 시 TEE가 키 방출(DP-07)·자격 발급(채널 상대 인증)을 거부하고, 채널 수립(DP-06)·정책 등록(DP-02)이 자격 부재로 거부된다. 검증 안 된 pVM은 "떠 있어도 아무것도 못 받는" 상태가 된다. 부팅 차단 가능 여부(H1 계약 연계)는 HV 확인 대상.
- **측정값 포맷(역반영 후보)**: 단계별 해시 누적(boot stage chain), manifest hash 포함, key id/version(DP-07 G4)·프로파일 선언(DP-08) 포함 — 채널 상대 인증 자격 발급과 DP-07 키 방출 대조에 동일 포맷 사용.
- **TA 예산**: 측정값 보관·대조 TA는 DP-03 G5 예산표 등록(저빈도 — 부팅·발급 시에만).
- **탐지·로깅 연계**: 검증 실패 이벤트는 보안 이벤트 로그 수집·격리 증빙 증빙 대상이다.

### 3.2 종합 판정

- **A: 최선의 기준안이지만 "pKVM 초기 측정 제공 및 범위" 확인 전에는 조건부.**
- **B**: 단독 부적합. 설치 시 조기 거부용 보조 검증으로만 사용 가능.
- **C**: 단순하지만 DP-07·DP-02가 요구하는 워크로드·자산 단위 바인딩을 닫지 못함.

### 3.3 권고안

후보 A를 조건부 채택하되 **HV 질의 게이트를 명확히 분리**해야 한다:

- pKVM 초기 측정은 DP-04의 H6(SMMU 의미론)과 운영상 같은 HV 협의 묶음으로 질의할 수는 있지만 **의미상 H6이 아니다**. H1 확장 또는 별도 **`H7: pVM 초기 측정/attestation 계약`**으로 두는 것이 맞다.
- **확정 가능 범위**: TEE가 키와 자격을 fail-closed로 거부한다, VMID 단독 인증 금지, 측정값 포맷은 manifest hash·profile·base/bootloader/kernel/payload hash·key epoch 포함.
- **확정 불가 범위**: pKVM이 실제로 무엇을 언제 측정해 TEE에 신뢰 경로로 전달하는지.

### 3.4 결함·누락·불일치 지적

1. **공통 부트로더 = 전 pVM 부팅 무결성의 공통 TCB.** 최소 기능, 코드 검증, rollback 방지, 버전별 측정값, 긴급 패치 경로를 결정문에 포함해야 함.
2. **부트로더→TEE 보고는 VMID만 믿으면 안 됨.** TEE nonce, EL2가 전달한 instance 식별자, manifest handle, measurement chain을 함께 묶어 replay/spoofing 차단 필요.
3. **fail-closed starvation은 "아무것도 못 받는다"가 아님.** 미검증 pVM은 여전히 CPU·메모리·VMM 자원을 점유하고 부채널은 공격 모델상 제외일 뿐 제거되지 않음. DP-04 자원 예산·강제 종료·보안 이벤트 로그 필요.
4. DP-03 G2는 A로 닫을 수 있으나 측정값을 채널 상대 인증 measured identity에 바인딩한 뒤 DP-03 경로에 역반영해야 닫힘.
5. 채널 상대 인증 설계은 TEE가 pVM ephemeral 공개키에 measurement-bound token을 서명하는 형태까지 연결해야 완결.
6. DP-07 G1/G2는 측정 종료점이 워크로드 코드·보호 자산까지 도달할 때만 닫힘 — 초기 pVM 이미지까지만이면 변조 payload 키 방출을 막지 못함.

## 4. 검수 (검수: Claude)

> 수행: critic 에이전트 (별도 컨텍스트, 전제 문서 8종 교차 대조, DP-08 문서와 합동 검수)

### 4.1 판정

**조건부 동의.** 후보 A 조건부 채택과 H7 분리 제안은 타당하며 DP-05의 H6 전례(사활적 HV 질의 분리)와 정합한다. 단 "지금 닫히는 범위 vs H7 확인 후 닫히는 범위"를 역반영 게이트별로 분리해 기록해야 하고, 부트스트랩 닭-달걀의 명시적 봉합이 필요하다.

### 4.2 주요 검수 의견

1. **Codex 등급·근거 타당.** 단 결함 3(미검증 pVM 자원 점유)은 새 결함이 아니라 **DP-01 6.3절 가용성(b) 비보장으로 이미 수용된 잔여 위험**이다 — DP-04 자원 회수 + 보안 이벤트 로그로 이관하면 충분하며 신규 게이트 불필요.
2. **H7 신설이 옳다(H1 확장 부적합).** H1은 lifecycle 계약이고 초기 측정은 측정값 생성·전달 계약으로 관심사가 다르다. 단 DP-04 G5 정본 규율에 따라 **H7 명세 보유는 DP-04 2절 정본**이어야 한다 — 본 DP는 H7 요구 명세(측정 대상 범위·TEE 신뢰 경로 전달 의미론·instance 식별자 동반)만 제공하고 정본 추가를 요청한다. H6과 같은 HV 세션에서 질의하되 폴백 트리거는 독립.
3. **(MAJOR) 부트스트랩 순환의 명시적 봉합** — DP-06 5절 순서의 (i)과 (ii) 사이에 "(i.5) 부트로더가 H7 경로(EL2 매개)로 측정값을 TEE에 **자격 없이** 보고" 단계가 필요하다. 측정 보고 경로(H7) ≠ 채널 상대 인증 credential buffer로 분리해야 "측정값 보고에 자격 필요 ↔ 자격 발급에 측정값 필요"의 순환이 끊긴다. **H7이 바로 이 순환을 끊는 메커니즘이다.**
4. **(MAJOR) 역반영 2단계 분리** — 측정값 포맷 골격 확정으로 지금 닫히는 것(채널 상대 인증 설계 claim 구조, DP-07 G1 대조 데이터 구조, DP-02 G5 참조 필드, DP-03 G2 방향)과 H7 확인 후에만 닫히는 것(측정 주체·전달 경로 실증 → DP-07 확정 전환, DP-03 G2 완전 폐쇄)을 구분해 기록한다.
5. **(MAJOR) anti-rollback 보강** — 측정값 포맷에 key epoch만으로는 부족하다. manifest version·base image version의 단조성을 TEE anti-rollback 카운터로 강제해야 구버전 패키지+구버전 key 동시 롤백을 막는다(DP-08 G6과 공동).
6. **부트로더 분류 정밀화** — 공통 부트로더는 per-pVM 인스턴스로 실행되는 게스트 코드로서 전역 TCB가 아니라 **"전 pVM 공통 측정 체인 루트 — EL2 초기 측정으로 무결성 보증, 침해 시 측정 보고 전체가 신뢰 불가가 되는 systemic 컴포넌트"**다. P1 베이스 이미지(P1 전용)와도 구분된다.

### 4.3 미해결 질문 (H7 질의 시 포함)

- pKVM 초기 측정의 측정 대상 granularity — "pVM 최초 메모리 이미지 전체"인지 "부트로더 영역만"인지에 따라 P1 베이스/P2 커널의 1차 측정 주체가 달라진다.
- P2 Secure OS의 자체 측정 루트와 본 체인이 이중이 될 때의 정본 선택(DP-08 G2 해소 과정에서 결정).

## 5. 결정

### 5.1 채택 구조

**후보 A를 조건부 채택한다 — "EL2 초기 측정 + pVM 내장 검증 부트로더 체인(pvmfw/DICE 패턴)".**

- 측정 체인: EL2 초기 측정(공통 부트로더) → 부트로더가 P1 베이스/P2 커널 검증·측정 → P1 베이스 런타임이 페이로드 + 보호 자산 chunk manifest 측정 → 누적 측정값을 H7 경로로 TEE 보고. **종료 지점 = 워크로드 코드 + 보호 자산 chunk manifest.**
- 검증 실패 시 차단은 **fail-closed starvation**: TEE가 키 방출(DP-07)·자격 발급(채널 상대 인증)을 거부하고 채널(DP-06)·정책 등록(DP-02)이 자격 부재로 거부된다. VMID 단독 인증 금지.
- **후보 B**는 설치 시 조기 거부용 보조 검증으로만 보존한다. **후보 C**는 H7 미제공 시의 폴백이다(payload-level 바인딩 미성립 한계 명시).
- 미검증 pVM의 자원 점유는 DP-01 6.3절 가용성(b) 비보장으로 기수용된 잔여 위험 — DP-04 자원 회수·강제 종료 + 보안 이벤트 로그로 이관.

### 5.4 상태

**조건부 채택** — G1(H7 HV 확인)이 사활적 게이트다. H7 확인 시 본 DP와 함께 DP-07이 확정으로 전환되고 DP-03 G2·DP-02 G5가 완전 폐쇄된다. H7 미제공 시 후보 C 폴백 + payload-level 보장 불가 한계를 전 문서에 연쇄 반영한다.
