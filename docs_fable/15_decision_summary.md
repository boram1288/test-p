## 3. 전 DP를 관통하는 공통 게이트

### 3.1 HV 파트 질의

- **H6 (SMMU 바인딩 제어 의미론)**: DP-02·04·05에 걸린 전역 최대 리스크. 최우선 단건 조기 질의 대상(DP-05 G1)이다.
- **H7 (pVM 초기 측정/attestation)**: DP-09 G1이 신설 요청하고 DP-04 2절 정본에 등재할 항목이다. DP-09·08, DP-03 G2, DP-02 G5에 걸린 두 번째 리스크 묶음이다.
- H7 확인 시 DP-09, DP-07, DP-03 G2, DP-02 G5의 측정값 연동 게이트를 완전 폐쇄할 수 있다.
- H7 미제공 시 DP-09 후보 C 폴백과 payload-level 보장 불가 한계를 전 문서에 반영해야 한다.

### 3.2 SRCX 협의

- TEE 신규 TA 자원 예산표(DP-03 G5): DP-07 key TA, DP-09 측정값 보관·대조 TA 등록 완료.
- pVM발 TEE 호출 DoS 쿼터·레이트리밋의 집행 주체(DP-03 G4, CONST-03 무회귀)를 협의해야 한다.

### 3.3 서비스 pVM TCB 편입 상한

- DP-07 후보 B(key broker pVM)는 기준안에서 배제한다. 도입하더라도 메타데이터·운영 보조로 한정하고, 최종 unwrap/release는 TEE가 수행한다.

## 4. 검수에서 발견되어 결정문에 반영된 핵심 사항

- **H7 신설(DP-09 검수)**: pKVM 초기 측정은 H1(lifecycle)도 H6(SMMU)도 아닌 별도 계약이다. 정본 보유는 DP-04, 요구 명세 제공은 DP-09가 담당한다.
- **DP-07 측정값 의존(CRITICAL)**: 키 방출 대상 판정은 DP-09 측정값에 의존한다. DP-07 골격은 DP-09 측정값 포맷과 전달 경로 역반영에 걸린다.
- **P2 보안 게이트 우회 경계(CRITICAL, DP-08 G2)**: 자체 키 관리·attestation을 가진 Secure OS가 DP-07 키 방출 게이트·DP-09 측정 체인을 우회하지 않을 의무를 conformance에 포함한다.
- **패키지 anti-rollback(DP-08 G6·DP-09 G3)**: 구버전 패키지 + 구버전 sealed key 동시 롤백 공격 차단을 위해 manifest version·base image version 단조성을 TEE anti-rollback 카운터로 강제한다.
- **crypto-erase 한계(DP-07)**: 실행 중 pVM의 키/평문은 DP-04 자원 회수 + DP-06 G4 채널 해제와 연동한다.

## 6. 남은 작업

1. HV 파트에 H6·H7 질의(동일 세션, 폴백 독립) — 회신 후 역반영 2단계 수행.
2. SRCX와 TA 예산표·DoS 쿼터 협의.
3. DP-04 G1: H1~H7 계약 명세 문서로의 승격.
4. DP-05 1단계 PoC(저빈도 경로) — H1·H2 + H6 시그니처 1차 고정과 병행.
5. 완결 선언은 위 1번 회신과 역반영 2단계 완료 후.
